# 背景

很多场景下需要服务能保证7*24h提供服务, 但是对于一个活跃的项目而言, 更新是家常便饭. 更新一般意味: 停止-启动服务, 这个停止-启动服务的时间窗口内就无法提供服务, 对于用户体验非常不好. 所以像Nginx这样的成熟项目都是支持gracefully restart. 普通项目也有gracefully restart的需求;

# 要求

  1. 升级过程不影响正在使用的用户;
  2. 升级后新接入的用户的流量被新启动的进程处理;
  3. …
